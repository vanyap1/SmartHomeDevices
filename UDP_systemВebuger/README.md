# Smart Home Monitor

Система моніторингу розумного дому з графічним інтерфейсом для відображення даних CAN шини.

## Структура проекту

- `smartHomeUdpService.py` - Базовий UDP клієнт для обробки CAN повідомлень
- `smart_home_data_classes.py` - Класи для структурованого парсингу даних
- `smart_home_gui.py` - Графічний інтерфейс користувача (PyQt5)
- `main_with_classes.py` - Консольна версія з новими класами
- `requirements.txt` - Залежності проекту

## Встановлення

1. Встановіть Python 3.7 або новіший
2. Встановіть залежності:
```bash
pip install -r requirements.txt
```

## Запуск

### Графічний інтерфейс
```bash
python smart_home_gui.py
```

### Консольна версія
```bash
python main_with_classes.py
```

## Опис даних

### Датчики температури (CAN ID: 0x69)
- Dallas температура
- Температура котла
- Температура теплоносія
- Температура сонячного колектора

### UPS система
#### Батарея (CAN ID: 0x33)
- Напруга (В)
- Струм (мА)
- Ємність (мАг)
- Статусні прапорці:
  - Джерело живлення
  - Стан помпи
  - Помилка котла
  - Помилка шлюзу
  - Низький заряд батареї
  - Перевантаження

#### Помпа (CAN ID: 0x43)
- Стан помпи
- Режим роботи

### Котел (CAN ID: 0x247)
- Режим роботи (Стоп/Передпуск/Робота/Помилка)
- Поточна температура
- Межі температури (верхня/нижня)
- Температури пуску/зупинки
- Дельта температури
- Статусні прапорці:
  - Реле 1
  - Статус реле
  - Датчик дверей

## Налаштування

Для зміни IP адреси шлюзу відредагуйте відповідні файли:
- В `smart_home_gui.py` змініть `gatewayIp="192.168.1.18"`
- В `main_with_classes.py` змініть `gatewayIp="192.168.1.18"`

## Особливості GUI

- Розмір вікна: 800x600 пікселів
- Темна тема з кольоровим кодуванням статусів
- Автоматичне оновлення даних в реальному часі
- Журнал подій з часовими мітками
- Індикатори стану з'єднання
- Лічильник отриманих пакетів

## Кольорове кодування

- Зелений - норма/активно
- Червоний - помилка/неактивно
- Жовтий - попередження/заголовки
- Блакитний - значення даних
